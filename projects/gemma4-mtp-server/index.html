Gemma4 Swift MTP Server: MLX 모델을 OpenAI 호환 API로 연결하는 Swift 서버

Gemma4 Swift MTP Server는 Apple Silicon의 MLX 모델을 기존 OpenAI 호환 클라이언트에서 호출할 수 있도록 만든 Swift 서버입니다. Gemma4 MTP와 Qwen DFlash draft path를 함께 다루고, streaming 응답과 on-demand backend 실행까지 한 흐름으로 묶었습니다.

마주한 문제 무거운 로컬 모델과 기존 클라이언트 사이의 간격

로컬 MLX 모델은 빠르게 실험하고 싶지만, 클라이언트는 대개 OpenAI 스타일 HTTP/SSE API를 기대합니다.

설계 방향 Swift 서버와 on-demand proxy 분리

추론 서버는 모델 로드와 생성에 집중하고, proxy는 요청이 있을 때 backend를 띄우고 idle 상태에서 정리합니다.

확인한 결과 draft token 검증이 포함된 로컬 API

MTP/DFlash 경로, acceptance length, reasoning split, HTTP/SSE 응답을 하나의 로컬 API로 실험할 수 있게 했습니다.

구현 포인트

  • Settings가 main model, draft model, token budget, verifier mode, benchmark mode를 CLI 옵션으로 받습니다.
  • generateMTPDetailed는 draft block, target verification, cache trimming, accepted token emission을 직접 관리합니다.
  • MTPHTTPServer는 /v1/models와 /v1/chat/completions를 처리하고 streaming chunk와 reasoning delta를 나눠 보냅니다.
  • Gemma4OnDemandProxy는 backend process 생명주기와 idle 종료를 관리해 로컬 메모리 점유를 줄입니다.

왜 중요한가

로컬 LLM은 모델만 있어서는 제품 흐름으로 들어오기 어렵습니다. API, streaming, 프로세스 수명, 메모리 비용까지 맞아야 실사용 도구와 연결됩니다.

어떻게 읽으면 좋은가

서버 시작, 모델 로드, MTP 생성 함수, HTTP handler, SSE response 순서로 읽으면 모델 추론이 API 응답으로 바뀌는 경로가 보입니다.

엔지니어링 관점

모델 내부 알고리즘과 제품 호환 API를 동시에 다룬 프로젝트입니다. 성능 실험을 기존 도구가 이해할 수 있는 인터페이스로 감싸는 능력을 보여줍니다.
