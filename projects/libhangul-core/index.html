LibHangul Swift/Linux: 한글 입력 상태와 한자 검색을 Swift로 정리한 코어

LibHangul Swift/Linux는 한글 입력기의 핵심이 되는 초성, 중성, 종성 조합과 한자 후보 검색을 Swift로 다루는 프로젝트입니다. PriType-Swift 같은 입력기 앱이 의존할 수 있도록 동기식 IME 콜백, Unicode 정규화, backspace edge case까지 고려했습니다.

마주한 문제 한글 조합은 단순 문자열 추가가 아닌 상태 문제

초중종성, 복합 모음, 복합 받침, backspace 분해, 영문/기호 커밋이 한 버퍼 안에서 충돌할 수 있습니다.

설계 방향 입력 context와 buffer 책임 분리

public context는 입력 흐름을 담당하고, buffer는 자모 조합과 분해 규칙을 맡도록 나누었습니다.

확인한 결과 IME 콜백에 맞는 동기 thread-safe API

OSAllocatedUnfairLock 기반 wrapper와 테스트 묶음으로 동기 입력기 환경에서 쓸 수 있는 형태를 갖췄습니다.

구현 포인트

  • HangulInputContext.swift는 KeyInput, Result 기반 처리, preedit/commit 문자열, 자판 전환, output mode를 관리합니다.
  • HangulBuffer.swift는 초성, 중성, 종성 상태와 복합 모음/받침 조합 및 backspace 분해를 담당합니다.
  • ThreadSafeHangulInputContext.swift는 actor 대신 OSAllocatedUnfairLock으로 동기 IME 콜백과 thread safety를 함께 맞춥니다.
  • HanjaTrie.swift는 sorted child array와 binary search로 한자 후보 검색의 힙 할당과 탐색 비용을 줄입니다.

왜 중요한가

입력기는 사용자가 매일 반복해서 쓰는 가장 민감한 시스템 소프트웨어입니다. 작은 조합 오류나 backspace 오류도 곧바로 경험 문제로 드러납니다.

어떻게 읽으면 좋은가

Context, Buffer, Character, Keyboard, Hanja 순서로 읽으면 입력 이벤트가 어떤 상태 변화를 거쳐 commit 문자열과 후보 검색으로 이어지는지 보입니다.

엔지니어링 관점

국문 입력이라는 도메인 지식을 자료구조와 동시성 API로 옮긴 프로젝트입니다. 플랫폼 콜백의 현실적인 제약을 고려해 라이브러리 경계를 잡았습니다.
