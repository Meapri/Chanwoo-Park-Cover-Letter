PriType-Swift: 입력 조합 상태를 직접 다루는 macOS 한글 IME

PriType-Swift는 macOS InputMethodKit과 libhangul을 연결해 한글 조합, 한자 후보, 입력 모드 상태를 관리하는 네이티브 입력기입니다. 입력기는 작은 상태 불일치도 바로 사용자의 불편으로 이어지기 때문에, 앱별 예외와 조합 생명주기를 명확하게 나누는 데 집중했습니다.

마주한 문제 앱마다 달라지는 입력기 이벤트와 커서 좌표

한글 조합은 키 입력, 후보창, 커서 좌표, text client 차이가 맞물려 작은 예외도 입력 깨짐으로 이어집니다.

설계 방향 IMK 이벤트와 libhangul 조합 상태 분리

입력 모드의 기준점을 한곳에 두고, force commit과 후보창 표시를 별도 흐름으로 분리해 추적하기 쉽게 만들었습니다.

확인한 결과 Chromium/Electron 예외를 별도 경로로 관리

일반 Cocoa 앱뿐 아니라 좌표와 delegate 동작이 다른 앱군을 감지하고 fallback 경로로 다룹니다.

구현 포인트

  • HangulComposer.swift에서 libhangul 조합 생명주기와 입력 모드의 기준 상태를 추적합니다.
  • main.swift에서 IMKServer 초기화, 한자 사전 preload, update check 흐름을 구성했습니다.
  • Chromium/Electron delegate와 cursor rect 예외, force commit 처리를 분리해 입력 상태가 꼬이는 범위를 줄였습니다.

살펴볼 파일

  • Sources/PriType/main.swift
  • Sources/PriTypeCore/HangulComposer.swift
  • Tests/PriTypeCoreTests

왜 중요한가

입력기는 사용자가 매 순간 체감하는 시스템 소프트웨어입니다. 한 글자가 늦게 커밋되거나 후보창 위치가 흔들려도 바로 불편해집니다. 이 프로젝트는 OS 프레임워크와 언어 조합 엔진 사이의 경계를 직접 다룹니다.

어떻게 읽으면 좋은가

main.swift에서 IMKServer와 초기화 흐름을 보고, HangulComposer.swift에서 조합 상태 전이를 따라가면 좋습니다. 한자 후보와 상태 UI는 별도 매니저와 윈도우 계층으로 분리되어 있습니다.

엔지니어링 관점

Apple platform API, C 라이브러리 연동, 사용자 입력 UX, 앱별 호환성 문제를 동시에 다룬 경험입니다. 재현하기 어려운 OS-앱 경계 문제를 좁혀 가는 방식이 드러납니다.
