PriType-Swift: macOS 입력기 경계 조건까지 다루는 한글 IME

PriType-Swift는 InputMethodKit과 libhangul을 연결해 한글 조합, 한자 후보, 상태 관리, 업데이트 흐름을 구성한 macOS 입력기입니다. 특히 Chromium/Electron 기반 앱에서 나타나는 delegate 수명과 커서 좌표 문제를 별도 경로로 분리해 입력 안정성을 높이는 방향으로 설계했습니다.

문제 영역 macOS IME의 조합 상태와 앱 호환성

한글 조합은 키 입력, 후보창, 커서 좌표, 앱별 text client 차이가 모두 맞물려 작은 예외도 입력 깨짐으로 이어집니다.

설계 초점 InputMethodKit과 libhangul의 책임 분리

IMK 이벤트 흐름과 libhangul 조합 상태를 분리해 입력 모드, force commit, 후보창 표시를 추적하기 쉽게 만들었습니다.

확인 포인트 Chromium/Electron 예외까지 코드화

일반 Cocoa 앱뿐 아니라 좌표와 delegate 동작이 다른 앱군을 별도 fallback 경로로 다룹니다.

구현 포인트

  • main.swift의 IMKServer 흐름과 libhangul 조합 상태를 분리해 입력 모드와 조합 라이프사이클을 추적하기 쉽게 만들었습니다.
  • 한자 사전 preload, 후보창, Right Option/Right Command 트리거를 입력기 경험 안에 연결했습니다.
  • Chromium/Electron 예외, cursor rect cache, force commit 처리를 별도 경로로 관리해 호환성 문제를 좁혔습니다.

왜 이 레포가 중요한가

입력기는 사용자가 가장 민감하게 체감하는 시스템 소프트웨어입니다. 한 글자가 늦게 커밋되거나 후보창 위치가 흔들려도 바로 불편해집니다. 이 레포는 OS 프레임워크와 언어 조합 엔진 사이의 경계 조건을 직접 다룹니다.

구조를 읽는 방법

main.swift에서 IMKServer와 초기화 흐름을 잡고, HangulComposer.swift가 조합 상태의 중심이 됩니다. 한자 후보와 상태 UI는 별도 매니저와 윈도우 계층으로 분리되어 입력 흐름을 침범하지 않게 구성되어 있습니다.

채용 관점의 의미

Apple platform API, C 라이브러리 연동, 사용자 입력 UX, 앱별 호환성 문제를 동시에 다룬 경험입니다. 운영체제와 맞닿은 문제를 구조적으로 좁혀 가는 방식이 드러납니다.
