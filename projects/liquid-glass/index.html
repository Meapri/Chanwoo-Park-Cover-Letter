liquid-glass-web: DOM 요소를 유리 소재처럼 다루는 렌더링 엔진

liquid-glass-web은 이 포트폴리오의 카드와 버튼에도 적용된 웹용 Liquid Glass 엔진입니다. 단순히 투명도를 올리는 대신, SVG filter와 WebGL refraction, displacement/specular map을 조합해 요소의 크기와 역할에 맞는 질감을 만듭니다.

마주한 문제 blur만으로는 유리처럼 느껴지지 않는 UI

배경을 흐리게 덮는 효과만으로는 곡률, 가장자리 굴절, 반사광이 부족해 실제 소재처럼 읽히기 어렵습니다.

설계 방향 요소 의미에 맞는 optical profile

bar, control, card, panel, selection을 크기와 역할로 분류하고 각 요소에 어울리는 질감과 품질을 선택합니다.

확인한 결과 성능과 fallback을 포함한 페이지 적용

DeviceProfile, lazy init, WebGL refractor, SVG fallback을 함께 두어 데스크톱과 모바일 환경을 모두 고려했습니다.

구현 포인트

  • AutoProfile.ts에서 요소의 크기, radius, semantic role을 읽어 알맞은 glass profile을 선택합니다.
  • FilterChain.ts에서 feImage, feDisplacementMap, specular map을 조합해 가장자리 굴절을 만듭니다.
  • DeviceProfile.ts와 fallback 흐름으로 기기 품질에 맞춰 효과를 조절하고 페이지가 무겁게 느껴지지 않게 했습니다.

살펴볼 파일

  • src/core/LiquidGlass.ts
  • src/core/FilterChain.ts
  • src/core/AutoProfile.ts

왜 중요한가

시각 효과는 보기 좋아야 하지만, 실제 제품에서는 성능과 접근성, fallback까지 함께 고려해야 합니다. 이 프로젝트는 디자인 의도를 브라우저 제약 안에서 재사용 가능한 엔진으로 바꾸는 작업입니다.

어떻게 읽으면 좋은가

LiquidGlass.ts에서 옵션과 생명주기를 보고, FilterChain.ts와 map cache 계층에서 SVG filter와 displacement/specular 이미지가 어떻게 연결되는지 따라가면 좋습니다.

엔지니어링 관점

렌더링 품질, GPU 비용, 접근성 설정, 모바일 품질 저하 조건을 함께 다룬 경험입니다. UI 감각을 실제 브라우저 엔진 구현으로 연결하는 문제 해결 방식을 보여줍니다.
