Event Log Pipeline: 재현 가능한 이벤트 생성부터 분석 대시보드까지

Test_PCW_2026은 이커머스 이벤트를 생성하고 PostgreSQL에 적재한 뒤 SQL 집계와 시각화까지 자동으로 수행하는 데이터 파이프라인입니다. Docker Compose로 실행 순서를 고정하고, seed와 기준 시간을 제어해 같은 결과를 반복해서 만들 수 있게 구성했습니다.

마주한 문제 데이터 파이프라인은 재실행 가능해야 합니다

분석 결과가 매번 달라지거나 DB 준비 전에 앱이 실행되면, 작은 과제라도 검증 가능한 결과물이 되기 어렵습니다.

설계 방향 deterministic generation과 idempotent load

seed, BASE_TIME, TRUNCATE, batch insert, healthcheck를 조합해 반복 실행 가능한 흐름을 만들었습니다.

확인한 결과 분석 쿼리와 dashboard export 자동화

이벤트 타입, 시간대별 트래픽/매출, 상위 사용자 분석을 한 번의 compose 실행으로 이미지까지 생성합니다.

구현 포인트

  • generator.py는 event weight, user pool, Faker seed, BASE_TIME을 통해 재현 가능한 로그를 만듭니다.
  • db.py는 DATABASE_URL 기반 연결, 재시도, TRUNCATE, batch insert로 적재 흐름을 안정화합니다.
  • analyzer.py는 세 가지 SQL aggregate를 DataFrame으로 반환하고, visualizer.py는 dashboard 이미지를 저장합니다.
  • docker-compose.yml은 PostgreSQL healthcheck와 depends_on: service_healthy로 실행 순서를 보장합니다.

살펴볼 파일

  • app/generator.py
  • app/db.py
  • app/analyzer.py
  • db/init.sql
  • docker-compose.yml

왜 중요한가

데이터 파이프라인은 화려한 모델보다 재현성, 멱등성, 스키마, 실행 순서가 먼저입니다. 작은 프로젝트 안에 그 기본기를 명확히 담았습니다.

어떻게 읽으면 좋은가

generator, db, analyzer, visualizer, compose 순서로 보면 데이터가 만들어지고 저장되고 분석되어 산출물로 남는 전체 경로를 볼 수 있습니다.

엔지니어링 관점

환경 변수, healthcheck, idempotent load, SQL index, dashboard export를 함께 다룬 실무형 과제입니다. 작은 규모에서도 검증 가능한 흐름을 만드는 태도를 보여줍니다.
