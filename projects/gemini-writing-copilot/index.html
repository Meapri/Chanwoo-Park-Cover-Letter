Gemini Writing Copilot: 코드 작업과 글쓰기 작업을 분리한 Codex 플러그인

Gemini Writing Copilot은 Codex 안에서 필요한 README, 포트폴리오 문구, 릴리즈 노트, 윤문 작업을 Gemini/Antigravity로 넘기는 개인 플러그인입니다. 개발 판단은 Codex에 남기고, 문장 작성은 별도 provider와 품질 검사를 거치도록 나누어 작업 흐름을 정리했습니다.

마주한 문제 코딩 맥락과 글쓰기 맥락이 섞이는 상황

구현 판단과 문장 생성이 같은 흐름에 섞이면 문구 품질과 근거 관리가 흔들리기 쉽습니다.

설계 방향 작업 유형별 prompt routing

README, release note, polish, product copy처럼 목적이 다른 글쓰기 요청을 task와 profile로 분리했습니다.

확인한 결과 로컬 컨텍스트를 살린 글쓰기 파이프라인

project context, strict source, quality gate를 함께 두어 코드 근거가 있는 문구를 만들 수 있게 했습니다.

구현 포인트

  • scripts/gemini_write.py에서 task, profile, output mode, project context, quality gate를 받아 하나의 writing prompt로 조립합니다.
  • antigravity_cli.py는 agy --print 호출만 담당하고, 로그인과 토큰 관리는 Antigravity CLI에 맡겨 인증 책임을 분리했습니다.
  • routing.py는 요청 문구를 읽어 README, release notes, polish 등 작업 유형과 기본 profile을 결정합니다.

살펴볼 파일

  • scripts/gemini_write.py
  • vendor/gemini_web_minimal/antigravity_cli.py
  • vendor/gemini_web_minimal/routing.py
  • tests/test_gemini_web_minimal.py

왜 중요한가

AI 도구를 제품처럼 쓰려면 모델 호출보다 작업 경계가 더 중요합니다. 이 프로젝트는 글쓰기 요청을 독립된 경로로 보내고, 근거와 품질을 다시 확인하는 작은 운영 도구입니다.

어떻게 읽으면 좋은가

CLI 인자 파싱에서 prompt 생성, provider 선택, 품질 경고 출력까지 이어지는 순서로 보면 플러그인이 어떤 책임을 갖는지 빠르게 파악할 수 있습니다.

엔지니어링 관점

작은 자동화라도 인증, provider fallback, secret redaction, testable routing을 나누면 오래 쓰기 쉬워집니다. 개인 도구를 재사용 가능한 플러그인으로 다듬은 경험입니다.
