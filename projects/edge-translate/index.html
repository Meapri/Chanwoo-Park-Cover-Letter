EdgeTranslate-v3: 실제 웹 페이지 번역을 견디는 MV3 라우팅

EdgeTranslate-v3는 페이지 번역을 단순 API 호출로 보지 않고, 브라우저 확장 안에서 여러 번역 경로와 UI 상태를 안정적으로 묶는 프로젝트입니다. Google/Page Translate, AI Page Translate, Gemini Nano, Chrome Built-in, Local/OpenAI 호환 경로를 분리해 라우팅하고, Manifest V3 서비스 워커 환경에서도 IndexedDB 캐시와 Preact 패널 상태가 끊기지 않도록 설계했습니다.

문제 영역 브라우저 번역 UX의 상태 불안정성

서비스 워커 수명, 페이지 DOM 변경, 스트리밍 응답, 패널 위치 상태가 동시에 움직이는 환경을 다룹니다.

설계 초점 번역 엔진 라우팅과 캐시 경계

번역 provider를 배경 스크립트에서 분리하고 IndexedDB 캐시를 붙여 반복 요청과 fallback 비용을 줄이는 구조입니다.

확인 포인트 실제 사용 흐름까지 이어지는 코드

라우팅, 캐시, content panel이 한 흐름으로 연결되어 번역 결과가 사용자 화면까지 자연스럽게 이어집니다.

구현 포인트

  • pageTranslate.js에서 웹 페이지 번역, AI 번역, 로컬 호환 API, Chrome Built-in 경로를 같은 흐름에서 선택할 수 있게 구성했습니다.
  • IndexedDB 기반 영구 캐시, URL 해시 프리패치, TTL 정리를 붙여 MV3 제약 안에서도 반복 번역 비용을 줄였습니다.
  • Preact 패널은 streaming result, floating/fixed 모드, docking, drag velocity, TTS까지 실제 사용 장면에 맞춰 다듬었습니다.

코드 근거

  • packages/EdgeTranslate/src/background/library/translationPersistentCache.js

왜 이 레포가 중요한가

페이지 번역은 텍스트를 API에 보내는 기능처럼 보이지만, 실제 브라우저에서는 DOM 분할, 언어 감지, 캐시, 모델 선택, UI 반응성이 함께 얽힙니다. 이 레포는 그 흐름을 기능별 경계로 나눠 확장 프로그램 구조 안에 묶은 작업입니다.

구조를 읽는 방법

pageTranslate.js에서 번역 경로를 고르고, translator 계층이 외부 또는 로컬 모델과 통신하며, content panel이 결과를 사용자에게 보여줍니다. 배경 스크립트와 콘텐츠 스크립트의 책임이 분리되어 흐름을 따라가기 쉽습니다.

채용 관점의 의미

브라우저 API 제약, 프론트엔드 인터랙션, 비동기 네트워크, 캐시 설계를 한 제품 안에서 함께 다룬 경험을 보여줍니다. 화면에 보이는 기능과 내부 상태 관리가 함께 안정적이어야 한다는 관점이 드러납니다.
