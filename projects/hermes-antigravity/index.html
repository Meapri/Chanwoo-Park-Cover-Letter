Hermes Antigravity: AI provider의 복잡한 경계를 정리하는 어댑터

hermes-google-antigravity-plugin은 Google Antigravity provider를 Hermes 흐름에 붙이기 위한 어댑터입니다. OAuth/gRPC 경계, Claude-style 도구 스키마, streaming event, tool-call id, context compression처럼 실패하기 쉬운 부분을 한 계층에서 정규화해 에이전트 세션이 예측 가능하게 이어지도록 만들었습니다.

문제 영역 AI provider 규격과 도구 호출의 불일치

인증, 스트리밍 이벤트, tool-call id, 모델 catalog가 서로 다른 규격으로 움직일 때 세션 흐름이 쉽게 끊어집니다.

설계 초점 어댑터 경계에서의 정규화

Claude schema와 Antigravity provider 응답을 내부 흐름에 맞게 정리하고, 도구 호출 생명주기를 한 계층에서 관리합니다.

확인 포인트 세션 안정성과 패치 검증

stable session id, orphaned tool cleanup, signature verification을 통해 provider 확장이 예측 가능하게 동작하도록 했습니다.

구현 포인트

  • google_antigravity_adapter.py에서 ProjectContext와 stable session id를 기준으로 모델 응답과 도구 호출 상태를 이어 붙였습니다.
  • Claude schema 정규화, tool config validation, tool-call id handling, orphaned tool part cleanup을 adapter 경계에서 처리했습니다.
  • provider registry, model picker, WebUI catalog 패치를 signature verification 흐름 위에서 적용하도록 구성했습니다.

코드 근거

  • agent/google_antigravity_adapter.py
  • patches/antigravity_provider_patch.py
  • plugins/model-providers/google-antigravity/plugin.yaml

왜 이 레포가 중요한가

AI agent 환경에서는 모델 응답만큼 도구 호출과 스트리밍 이벤트의 안정성이 중요합니다. 이 레포는 서로 다른 provider 규격을 한 흐름으로 연결하면서, 실패하기 쉬운 세션 상태와 tool-call 파편을 코드에서 정리한 작업입니다.

구조를 읽는 방법

google_antigravity_adapter.py에서 provider 응답을 내부 이벤트로 바꾸는 흐름을 먼저 보면 좋습니다. 이후 patch 계층과 plugin yaml을 따라가면 모델 선택기, WebUI catalog, provider registry가 어떻게 연결되는지 확인할 수 있습니다.

채용 관점의 의미

AI 기능을 붙이는 수준을 넘어 인증, 이벤트 변환, 스키마 호환성, 세션 안정성을 함께 다룬 경험입니다. 빠르게 변하는 provider 생태계에서도 경계를 읽고 안정적인 어댑터를 설계하는 역량을 보여줍니다.
