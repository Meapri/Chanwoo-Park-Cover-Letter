Hermes Antigravity: 서로 다른 AI provider 규격을 이어 주는 어댑터

hermes-google-antigravity-plugin은 Google Antigravity provider를 Hermes 흐름에 붙이기 위한 Python 어댑터입니다. 인증, streaming event, tool-call id, Claude-style schema처럼 provider마다 달라지는 경계를 한 계층에서 정규화해 세션 흐름을 안정적으로 유지하는 데 집중했습니다.

마주한 문제 provider마다 달라지는 응답과 도구 호출 규격

인증, streaming event, tool-call id, model catalog가 서로 다른 규격으로 움직이면 agent 세션이 쉽게 끊어집니다.

설계 방향 어댑터 경계에서 schema와 이벤트 정리

provider 응답을 내부 이벤트로 바꾸고, tool config와 tool-call 생명주기를 한 계층에서 관리했습니다.

확인한 결과 세션 안정성과 patch 검증 흐름

stable session id, orphaned tool cleanup, signature verification을 통해 provider 확장이 예측 가능하게 동작하도록 했습니다.

구현 포인트

  • google_antigravity_adapter.py에서 ProjectContext와 stable session id를 기준으로 응답과 도구 호출 상태를 이어 붙였습니다.
  • Claude schema normalization, tool config validation, tool-call id handling, orphaned tool part cleanup을 adapter 경계에서 처리했습니다.
  • provider registry, model picker, WebUI catalog patch를 signature verification 흐름 위에서 적용하도록 구성했습니다.

살펴볼 파일

  • agent/google_antigravity_adapter.py
  • patches/antigravity_provider_patch.py
  • plugins/model-providers/google-antigravity/plugin.yaml

왜 중요한가

AI agent 환경에서는 모델 응답 자체만큼 도구 호출과 스트리밍 이벤트의 안정성이 중요합니다. 이 프로젝트는 서로 다른 provider 규격을 한 흐름으로 연결하며, 실패하기 쉬운 세션 상태와 tool-call 파편을 정리한 작업입니다.

어떻게 읽으면 좋은가

google_antigravity_adapter.py에서 provider 응답을 내부 이벤트로 바꾸는 흐름을 먼저 보면 좋습니다. 이후 patch 계층과 plugin yaml을 따라가면 모델 선택기, WebUI catalog, provider registry의 연결이 보입니다.

엔지니어링 관점

AI 기능을 붙이는 수준을 넘어 인증, 이벤트 변환, schema 호환성, 세션 안정성을 함께 다룬 경험입니다. 빠르게 변하는 provider 생태계에서 경계를 읽고 안정적인 어댑터를 설계하는 역량을 보여줍니다.
