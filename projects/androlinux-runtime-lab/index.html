androlinux-runtime-lab: Android 샌드박스 안에서 Linux 실행 환경을 검증하는 랩

androlinux-runtime-lab은 Android 앱 내부에서 Linux rootfs와 native runtime을 다루기 위한 실험 프로젝트입니다. 설치, 검증, 실행, GPU 진단을 각각 나누어 앱 전용 저장소의 제약과 NDK 실행 경로를 안전하게 확인했습니다.

마주한 문제 앱 전용 공간의 실행 제약

rootfs를 내려받아도 Android의 보안 모델 안에서는 경로, 권한, loader, native library 위치를 모두 따져야 합니다.

설계 방향 검증된 rootfs와 packaged backend

manifest 검증 후 추출하고, rootfs 실행은 직접 실행이 아니라 packaged PRoot와 loader 계획으로 분리했습니다.

확인한 결과 CLI probe와 GPU probe가 있는 런타임 랩

dpkg/apt probe, linker64 probe, EGL/GLES pbuffer와 surface render 진단까지 묶어 실행 가능성을 단계별로 확인했습니다.

구현 포인트

  • RootfsInstaller.kt는 SHA-256과 size를 확인한 뒤 tar entry의 절대 경로, 상위 경로 이동, 링크, device entry를 막고 추출합니다.
  • NativeCommandRunner.kt는 packaged PRoot, Android linker, apt/dpkg, GPU/GUI bridge probe를 명확한 실행 함수로 나눴습니다.
  • runtime_report.cpp는 EGL/GLES pbuffer와 Android surface render를 통해 host GPU와 software renderer 여부를 진단합니다.

왜 중요한가

Android에서 Linux 런타임을 다루는 일은 단순 실행보다 경계 관리가 중요합니다. 이 프로젝트는 파일 시스템, loader, 환경 변수, graphics probe를 서로 다른 책임으로 나눕니다.

어떻게 읽으면 좋은가

rootfs 설치 계획에서 시작해 native command runner, C++ runtime report 순서로 보면 앱 안의 Linux 실행이 어떤 단계로 검증되는지 자연스럽게 이어집니다.

엔지니어링 관점

위험한 압축 해제와 네이티브 실행을 작은 검증 단위로 쪼갠 점이 핵심입니다. 플랫폼 제약을 우회하지 않고 진단 가능한 구조로 바꾸는 경험을 보여줍니다.
